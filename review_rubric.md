# 코드 품질 리뷰 루브릭

Claude Code가 `/review` 실행 시 이 루브릭에 따라 코드를 평가한다.
**감점 지적만 출력한다. 잘된 점, 칭찬, 설명은 출력하지 않는다.**

---

## 평가 항목

### 1. 알고리즘 선택 (Algorithm)
- 문제를 해결할 수 있는 더 단순하거나 효율적인 방법이 있는데 과도하게 복잡한 방식을 선택했는가?
- 루프 구조가 불필요하게 중첩되어 있는가?
- `pow()` 대신 직접 반복 곱셈이 필요한 문제에서 `pow()`만 사용했는가?

### 2. 가독성 (Readability)
- 의미 없는 단일 문자 변수명을 사용했는가? (`a`, `b`, `t`, `s` 등 — `i`, `j`, `k`, `n`, `x`, `y`는 관용적으로 허용)
- 매직 넘버를 직접 사용했는가? (의미가 명확하지 않은 숫자 상수)
- 한 함수(main)에 모든 로직이 집중되어 100줄을 초과하는가?

### 3. 코딩 스타일 (Style)
- 들여쓰기가 불일관적인가? (탭/스페이스 혼용)
- `void main()` 을 사용했는가?
- `main` 함수에 `return` 문이 없는가?
- 주석이 전혀 없어 코드 의도를 파악하기 어려운가?

### 4. 메모리/리소스 (Memory)
- 배열 범위를 초과하는 접근이 의심되는가?
- 동적 할당 후 해제하지 않는 명백한 패턴이 있는가?
- 불필요하게 큰 배열을 선언했는가?

### 5. 입출력 처리 (I/O)
- 입력 검증 없이 잘못된 입력에 취약한 구조인가?
- 출력 형식이 문제 요구사항과 명백히 다른가?

---

## 출력 형식

```
[학번] 홍길동
aiFeedback: (한 줄 요약 — 가장 심각한 문제 1~2개)
suggestions:
- 구체적인 개선 제안 (파일명:줄번호 포함)
- ...
```

감점 사유가 없으면:
```
[학번] 홍길동
aiFeedback: (없음)
suggestions:
```

---

## 주의사항

- 각 학생에 대해 **한 번만** 판단한다. 중복 분석하지 않는다.
- 리뷰는 채점 점수에 **반영되지 않는다** — 학습 피드백 전용이다.
- `criteria.md`의 채점 항목과 중복되어도 무방하다 (관점이 다름).
